JavaScript 模块儿化

模块儿化的概念解读

  1. 模块儿化起源与NodeJS，但是这是后端的，
    1. 后来在前端也出现了两个模块儿化的JS
    2. 国外的ReqiureJS、CommonJS和国内的Seajs
  2. AMD 是RequireJS 在推广过程中对模块儿定义的规范化产出
    1. 异步加载模块儿，依赖前置，提前执行。
    2. Define定义模块儿
       define(['require','foo'],function(){return});
    3. Require加载模块儿（依赖前置）
       require(['foo','bar'],function(foo,bar){});
  3. CMD 是SeaJS在推广过程中对模块儿定义的规范产出。
    1. Define定义 exports导出,module上存储了当前模块儿上的一些对象。
       define(function(require,exports,module){});
    2. require(./a)直接引入。Require.async支持异步引入。
  4. 同步加载，依赖就近，延迟执行。
    1. 类似的还有CommonJS Modules/2.0规范。
  5. AMD 是预加载，CMD 是懒加载

Seajs的应用

官网：

  1. 官方文档：https://seajs.github.io/seajs/
  2. 入门网站：https://seajs.github.io/seajs/docs/#quick-start
  3. github里的 seajs库： https://github.com/seajs/seajs
  4. github里的 例子：https://github.com/seajs/examples

用法

  1. 参照nodejs的思想
  2. 首先 define(function(require,exports,module){})
    1. require可以像nodejs一样引入其它模块儿，
    2. 同步引入使用require，
    3. 异步引入使用require.async('模块儿名',function(obj){...})
  3. exports 可以通过 exports.成员名 的方式单个单个导出成员
  4. module 可以通过 module.exports={key:value,key2:value2} 的方式多导出成员

  5. 页面引入seajs之后，你可以通过

     <script type="text/javascript">
     // seajs 的简单配置
     //调用配置的方法
     seajs.config({
         //根目录
       base: "./js/",
       //起一个别名
       alias: {
         key : value
         "jquery": "jquery-debug.js"
       }
     })

     // 加载入口模块 main.js后面的.js会自动添加,这个main.js是根目录下的
     seajs.use("main")
 </script>

原理

  1. 通过script标签的src属性 去引入，
  2. 然后获取每一个js里面的内容，
  3. 然后解析内容字符串，
  4. 生成新的script，
  5. 嵌入到页面中，然后调用执行。

函数柯里化（curry）

  1. 柯里化，把接收多个参数的函数变换成接收一个单一的函数。
    1. 合并参数，第一个函数的参数合并成整体的，传递给统一的函数。
  2. function是js中的王者。

  3. 函数柯里化代码

     //创建柯里化函数
 function curry(fn){
     //获取函数以外的其它参数
     var args=Array.prototype.slice.call(arguments,1);
     //返回一个函数
     retrun function(){
         //获取所有参数
         var innerArgs=Array.prototype.slice.call(arguments);
         //拼接参数
         var finalArgs=args.concat(innerArgs);
         // 返回执行结果
         retrun fn.apply(this,finalArgs);
     }
 }

 //创建 最后执行的函数
 function add(num1,num2,num3) {
     retrun num1+num2+num3;
 } 
 // 利用了函数作用域链和闭包
 var t=curry(add,50)(1,2);

 alert(t);

JavaScript高级技巧

变量作用域和闭包

  1. js是函数级别作用域，
    1. 在内部的变量，
    2. 内部都可以访问，
    3. 外部的不能访问内部的，
    4. 内部可以访问外部。
  2. 变量提升、函数提升，
    1. 先变量提升，然后再函数提升。
  3. 闭包就是拿到本不该拿的东西，
    1. 容易操成内存泄漏，你使用闭包之后，
    2. 系统不知道你什么时候用，所以不会轻易地给你清除掉。
  4. Javascript在前端的内存泄漏不是很严重，
    1. 但是到了后端这个内存泄漏会凸显出来。

This指针和使用

  1. 正常情况下，谁调就会指向谁。
  2. 如果你使用了有关于bind函数，
    1. 那么你绑定谁，this就指向谁
  3. 找成员先找自身的，然后再去找原型中的

按值传递和按引用传递

  1. 值应用传递值
  2. 引用传递传递引用。

JavaScript面向切面编程

面向切面的概念解读

  1. JavaScript的最大的优点，
    1. 是没有任何束缚没有任何约定，
    2. 只需要按照编程的思想，
    3. 就能够实现所需要的东西。
  2. 使用随意的语法然后引用function，
    1. 就可以组装面向切面编程的方法。
  3. Aspect Oriented Programming(AOP),
    1. 面向切面编程，是一个比较热门的话题。
  4. AOP主要实现的目的是针对业务处理过程中的切面进行提取，
    1. 它所面对的是处理过程中某个步骤或者阶段，
    2. 以获得逻辑过程中各部分之间低耦合性的隔离效果。
  5. AOP 最大的优点是 可以对业务进行无侵入的干扰。

    // 使用 aop 来进行无侵入的实现

    function test () {
      alert(1);
    }

    // 给所有的函数扩展原型 before
    Function.prototype.before = function (fn) {
      // 记录自己
      var __self = this;
      // 先调用回调
      fn();
      // 然后调用自己 
      __self.apply(this,arguments);

    }

    // 给所有的函数扩展原型 after
    Function.prototype.after = function (fn) {
      var __self = this;
      // 先调用自己
      __self.apply(this, arguments);
      // 然后调用回调
      fn();

    }

    test.before(function () {
      alert(0);

    })

    test.after(function () {
      alert(2);

    })

    // 结论  默认函数 被执行了 两次 before 和 after 各执行一次

    function test () {

        }

      })

JavaScript多线程

多线程概念介绍

  1. 多线程，
    1. 是指从软件或者硬件上实现多个线程并发执行的技术。
  2. 具有多线程能力的计算机因有硬件支持而能够在同一时间执行多于一个线程（台湾译作执行绪），
    1. 进而提升整体处理性能。
  3. 具有这种能力的系统包括对称多处理机、
    1. 多核心处理器以及芯片级多处理或同事多线程处理器。
  4. 在一个程序中，
    1. 这些独立运行的程序片段叫 线程(Tread)，
    2. 利用它编程的概念就叫做 多线程处理（Multithreading）。
  5. js是单线程的，
    1. 但是分同步代码和异步代码，
    2. 异步代码会放到异步队列中去，
    3. 当同步代码执行完毕之后再去执行异步队列中的代码
  6. js中的异步代码有 事件、定时器、异步请求。

 onconnect = function(e) {
     //获取连接的端口
     var port = e.ports[0];
     var num = 0;
     var result = 0;
     port.postMessage("连接成功！");
     port.onmessage = function(e) {
         num=e.data;
         for (var i = 1; i <= num; i++) {
             result += i;
             //这个console.log(result);无效。
             // console.log(result);
             // 不停的返回数据
             port.postMessage(result);
         }
         port.postMessage(result);
     }
 }
